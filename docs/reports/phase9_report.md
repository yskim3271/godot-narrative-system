# Phase 9. 테스트 / QA 강화

## 목표
종단 통합 플로우 검증, 적대적 저장 데이터 강화 테스트, 그리고 "해피패스는 엔진 에러·경고 0건"을 기계적으로 보장하는 검증 파이프라인을 완성한다.

## 구현 내용
- **통합 플로우 테스트** (`test_integration_flow.gd`): 출하 데모 DB를 파사드 경유로 종단 재생 — 첫 대화→퀘스트 수주→3/5 진행→**선택지 화면 한가운데 저장**→두 번째 파사드("새 프로세스")로 로드→재개→진행 보고→완료(보상 130골드)→언어 전환→최종 저장/로드 왕복. 데모 컷신이 실제로 돌도록 미니 월드(액터+wave 애니메이션+카메라) 구성.
- **저장 강화 테스트** (`test_save_hardening.gd`, 6종): 섹션 타입 오염(dict 대신 문자열/배열/숫자/불) → 해당 섹션만 기본값, 내부 항목 오염(객체 대신 문자열 objective, 무효 상태, 잡탕 history, 비배열 seen) → 항목 단위 드롭, 잘린 파일 격리, **DB 기준 카운트 재클램프**(999→5/completed, -3→0), 제거된 퀘스트의 레거시 상태 보존, 반복되는 불량 조건의 안정성.
- **검증 파이프라인 완성** (`scripts/run_tests.ps1`, 4단계): import → 전체 유닛(+SCRIPT ERROR 0 게이트) → **해피패스 순수성 게이트**(통합 플로우 출력에 `ERROR|WARNING|SCRIPT ERROR` 라인이 하나라도 있으면 실패) → 데모 DB validator CLI `--strict`. 집계 exit code.

## 검증 방법 / 테스트 결과
- 최종 파이프라인: **ALL GREEN, exit 0** — 유닛 **137/137 (4.6s)**, SCRIPT ERROR 0, 해피패스 클린(에러·경고 0), 데모 DB strict 0/0.
- 파이프라인 2회 연속 실행 일관 (프레임 경계 고정·결정적 시그널 설계 덕에 플레이크 없음).

## 발견된 문제 (해결됨 — 순수성 게이트의 성과 2건)
1. 통합 테스트가 헤드리스에서 액터/카메라 없이 컷신을 실행해 "unknown actor" 경고 → 게이트가 차단. 실게임과 동일한 미니 월드 등록으로 해결 (테스트 충실도도 상승).
2. **파킹된 시퀀서 코루틴 누수**: 취소된 런도 `wait()` 타이머가 만료될 때까지 코루틴이 컨텍스트→DB→텍스처 RID를 붙잡음 — 짧은 필터 실행에서만 종료 시 누수 진단 3건 발생(전체 스위트에선 타이머가 먼저 만료돼 은폐). after_each에서 최장 wait 시간만큼 대기해 해소. 이 동작은 known_limitations에 문서화 예정(게임 종료 시 무해, 장수명 씬에서는 자연 해소).
